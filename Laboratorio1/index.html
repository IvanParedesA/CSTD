Preguntas

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • El internet es el medio, es decir la conexión que hay entre todas las computadoras, mientras que la World Wide Web es el conjunto de todos los sitios que existen en esta red.

  • ¿Cuáles son las partes de un URL?

    • De acuerdo con Edix (2022), las partes de un URL son las siguientes:

    • Esquema: es el protocolo de inicio empleado para la transmisión de datos entre ordenadores y la recuperación de la información de internet. En la mayor parte de los casos el esquema se corresponde con: http o https.
    • Host: normalmente, para el protocolo http, el host es: www.
    • Dominio: corresponde con el nombre del dominio de cada sitio web, como: edix.com.
    • Puerto: define el nombre del puerto según el host. Para el protocolo http, el puerto por defecto es 80, pero esta parte suele omitirse.
    • Directorio (o ruta): define el camino que se debe seguir en el servidor. Por ejemplo: blog.
    • Nombre de archivo: define detalladamente las características del documento. Por ejemplo: videos-gratis

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Con información de Williams Morales (2021), esta es la función de los diferentes métodos:

    • GET es para obtener un recurso (lo usan todos los hipervínculos). Retorna tanto la cabecera como el contenido
    • HEAD es muy similar a GET con la diferencia de que se utiliza solo cuanto se está interesado en el código de HTTP de Respuesta y el encabezado (Headers)
    • POST sí tiene body y este viene cifrado. Se usa cuando envías datos, por ejemplo contraseñas
    • PUT es similar al POST, pero este funciona para actualizar información existente.
    • PATCH igualmente es similar a los dos anteriores pero se utiliza para hacer moficaciones parciales de un recurso en particular.
    • DELETE se utiliza para eliminar un recurso en específico

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Se debe usar el método POST porque este cifra la información que mandas.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • El método GET

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • Al contrario, este código significa que todo funciona correctamente.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • No es responsabilidad del desarrollador ya que los codigos en el rango de los 400s son usualmente errores del cliente.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • En este caso si es responsabilidad del desarrollador porque los errores en el rango de los 500s indican un problema con el servidor web.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Significa que con la nueva actualización de html, este ha sido reemplazado por nuevos elementos que hacen lo mismo o mejoran las funciones. Aún así, muchos navegadores todavía soportan este tipo de atributos. A continuación algunos ejemplos:

    • applet: Reemplazado por el tag HTML object.
    • dir: Reemplazado por el tag HTML ul.
    • isindex: Reemplazado por el tag HTML input.
    • menu: Reemplazado por el tag HTML ul.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • Entre HTML4 y HTML5 es fácil darse cuenta de que algunos elementos y etiquetas tienen un nombre diferente o fueron sustituidas por otros atributos que hacen lo mismo pero con un cambio añadido. Sin embargo, otros cambios también están enfocados a elementos multimedia, seguridad y actualización, ya que no se tiene planeado un HTML6. Además, con las nuevas funcionalidades, HTML5 ya no es completamente dependiente de CSS o JavaScript.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • De acuerdo con (De León, s.f.) los componentes del elemento table son los siguientes:

    • caption: Provee un título para la tabla
    • colgroup: agrupa columnas temáticamente
    • col: aplica atributos comunes y estilos a una o más columnas. Sólo puede ser declarado como hijo de colgroup.
    • tbody: representa a un cuerpo de la tabla.
    • thead: representa la sección de encabezado de la tabla.
    • tfoot: representa la sección de pie de la tabla.
    • tr: representa una fila. Puede ser declarado como hijo de tbody, thead, tfoot o table.
    • td: representa a una celda regular. Sólo puede ser declarado como hijo de tr.
    • th: representa a una celda de encabezado. Sólo puede ser declarado como hijo de tr.

  • ¿Cuáles son los principales controles de una forma HTML5?

    • Los principales controles son los siguientes:

    • number: campo de entrada para la introducción de un número
    • range: permite elegir entre un rango de valores
    • date: selector de fecha
    • color: permite al usuario elegir entre una selección de colores
    • datalist: permite a los usuarios elegir entre una lista de opciones sugeridas
    • tel: campo de entrada para un número telefónico
    • email: campo de entrada para un email
    • url: campo de entrada para un url
    • text: campo de entrada para texto
    • time: campo de entrada para una hora específica

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    • El ciclo de vida consiste en los siguientes pasos:

    • Fase de planificación: se prepara el diseño y posterior implementación del sistema.
    • Fase de análisis: se busca concretar una serie de requisitos, que son los que regirán el nuevo sistema o los cambios a introducir en el antiguo.
    • Fase de diseño: se identifican soluciones potenciales, se evalúan y se elige la más conveniente.
    • Fase de desarrollo: Inicio de la producción.
    • Fase de integración y periodo de pruebas: corroborar que el diseño propuesto cumple con los requisitos de negocio establecidos.
    • Fase de implementación: instalación del hardware y software elegidos, crear las aplicaciones correspondientes, someterlas a pruebas, crear la documentación pertinente y capacitar a los usuarios.
    • Fase de mantenimiento: consiste en introducir los ajustes necesarios para mejorar el rendimiento y corregir los problemas que puedan surgir.
    • Retiro del software: LA última etapa de un sistema de información ocurre cuando es finalmente desmantelado y no se le da mantenimiento ni uso nunca más.

  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    • En pocas palabras, el ciclo de desarrollo es el mismo que el ciclo de vida de un sistema, pero sin incluir la parte del retiro del software.
